YACRS Configuration

YACRS (Yet Another Class Response System) is a simple application for recieving feedback from students during classroom teaching. The students use a browser on their own computers or mobile phones, and the teacher can either use a browser or a simple floating application to run the session.

Requirements

  • Web server with PHP 5 or later.
  • MySQL database
  • Access to an LDAP system for authentication

Installation

First create an empty MySQL database to be used by YACRS - at the command line log in to MySQL as the root user:  In this example the database, user and password are all 'yacrs', however these can be anything you wish. This example also assumes that the database server is on the same machine as the web server.

> sudo mysql -u root -p.

Type the password

mysql> CREATE DATABASE yacrs;
mysql> GRANT ALL PRIVILEGES ON yacrs.* TO 'yacrs'@'localhost' IDENTIFIED BY 'yacrs';
mysql> exit;

Next copy the YACRS disribution files into a folder in the web server folder, e.g. on Ubuntu 18.04 this might be /var/www/html/yacrs

You will need to either edit the config.php file, or create a configuration file called yacrs_config.php in the folder above it.

The best approach is to follow the instructions in the config_example.php file, modifying it to suit your needs. You will need to know the settings for your institutions LDAP server.

Once the configuration has been completed and saved, you can run initialise.php to initialise the database. (https://yourservername.edu/yacrs/initialise.pho)

Once that is done you should be ready to go!
